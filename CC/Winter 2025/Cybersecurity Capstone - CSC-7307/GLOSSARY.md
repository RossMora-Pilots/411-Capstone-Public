# Glossary of Cybersecurity Terms

> For readers unfamiliar with cybersecurity terminology. Terms are used throughout this portfolio.

---

| Term | Definition |
|------|-----------|
| **ACL** | Access Control List — a set of rules that filters network traffic by allowing or denying packets based on source/destination IP, port, or protocol |
| **Annex A** | The appendix of ISO/IEC 27001 containing 93 reference security controls organized across 4 themes (Organizational, People, Physical, Technological) |
| **CAB** | Change Advisory Board — the group responsible for evaluating and approving proposed changes to production systems |
| **CHR** | Cloud Hosted Router — MikroTik's virtual router appliance deployed as a VM |
| **CVSS** | Common Vulnerability Scoring System — a standardized framework for rating the severity of security vulnerabilities (0.0–10.0 scale) |
| **Decoder** | In Wazuh, an XML-defined pattern matcher that extracts structured fields from raw log messages |
| **GNS3** | Graphical Network Simulator-3 — an open-source network emulation platform used to run virtual Cisco routers |
| **GRC** | Governance, Risk, and Compliance — the integrated approach to managing organizational governance, enterprise risk, and regulatory compliance |
| **Hyper-V** | Microsoft's hardware virtualization platform for running virtual machines on Windows |
| **ISMS** | Information Security Management System — the framework of policies, procedures, and controls for managing information security risks (core concept of ISO 27001) |
| **ISO 27001** | ISO/IEC 27001:2022 — the international standard specifying requirements for establishing, implementing, maintaining, and improving an ISMS |
| **JSON** | JavaScript Object Notation — a lightweight, human-readable data format used for structured log data |
| **MIB** | Management Information Base — a database of objects that can be monitored via SNMP on a network device |
| **NTP** | Network Time Protocol — synchronizes system clocks across devices for accurate log correlation |
| **OpenNMS** | An open-source network monitoring platform; Industry Partner's existing monitoring infrastructure |
| **ossec.conf** | The primary Wazuh Manager configuration file located at `/var/ossec/etc/ossec.conf` |
| **PDCA** | Plan-Do-Check-Act — the continuous improvement cycle used in ISO 27001 |
| **RAG** | Red-Amber-Green — a status reporting system where Red = at risk, Amber = minor issues, Green = on track |
| **RBAC** | Role-Based Access Control — restricting system access based on organizational roles |
| **SIEM** | Security Information and Event Management — a platform that collects, correlates, and analyzes security events from multiple sources in real time |
| **SNMP** | Simple Network Management Protocol — a protocol for collecting device health data (CPU, memory, interface status) from network equipment |
| **SOC** | Security Operations Center — a centralized team that monitors, detects, and responds to cybersecurity incidents |
| **Syslog** | A standard protocol (typically UDP 514) for sending log messages from network devices to a central collector |
| **VM** | Virtual Machine — an isolated computing environment running on a hypervisor (e.g., Hyper-V) |
| **Wazuh** | An open-source SIEM platform providing log collection, threat detection, file integrity monitoring, and compliance reporting |
| **XML** | Extensible Markup Language — used by Wazuh for decoder and rule definitions |
| **xmllint** | A command-line XML validation tool used to check configuration file syntax before applying changes |

---

*Terms are defined in the context of this capstone project. For comprehensive definitions, see the [Wazuh Documentation](https://documentation.wazuh.com/) and [ISO/IEC 27001:2022](https://www.iso.org/standard/27001).*
